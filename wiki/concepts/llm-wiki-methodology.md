# LLM Wiki 方法论

一种利用 LLM 增量构建和维护持久化个人知识库的方法论。核心理念是让 LLM 充当知识库的"程序员"，人类担任"架构师"和"策展人"。

## 核心思想

传统 RAG（检索增强生成）系统在每次查询时从原始文档中重新发现知识，没有积累。LLM Wiki 则相反：LLM 读取源文件，提取关键信息，并整合到已有的维基中——更新实体页面、修正主题摘要、标注矛盾点。**知识被编译一次并持续更新**，而非每次查询时重新推导。

## 三层架构

### 原始源层 (`raw/`)
- 不可变的源文件集合（文章、论文、剪藏、PDF 等）
- LLM 只读不写
- 这是知识库的"真相来源"

### 维基层 (`wiki/`)
- LLM 生成和维护的 markdown 文件
- 包含：摘要、实体页面、概念页面、比较分析、综合综述
- LLM 负责创建、更新、维护交叉引用和一致性
- 人类读取；LLM 写入

### 模式层 (`schema/`)
- 配置文档（如 CLAUDE.md），定义：
  - 维基的结构和惯例
  - 摄取、查询和健康检查的工作流
- 这是让 LLM 成为有纪律的维基维护者的关键

## 三个核心操作

### Ingest（摄取）
新源文件加入后，LLM 执行：
1. 读取源内容
2. 与用户讨论要点
3. 创建源摘要页面
4. 更新相关概念页面
5. 更新索引
6. 记录日志

一个源可能触及 10-15 个维基页面。

### Query（查询）
LLM 搜索相关页面并合成带引用的答案。有价值的答案应归档回维基作为新的查询页面。

### Lint（健康检查）
定期检查：页面矛盾、过时声明、孤儿页面、缺失概念、缺失交叉引用。

## 特殊文件

- **`index.md`** — 内容导向的目录，按类别列出所有页面
- **`log.md`** — 时间顺序的操作记录

## 关键洞察

LLM Wiki 在精神上与 Vannevar Bush 1945 年的 Memex 构想相关——一个经过个人策划的知识存储，文档之间有联想路径。Bush 无法解决的问题是谁来做维护工作。**LLM 解决了这个问题。**
